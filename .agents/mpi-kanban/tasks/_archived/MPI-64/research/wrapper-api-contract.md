# Wrapper API Contract — RunPod Remote Engine

**Status:** Design decision record. Consumes `transport-topology.md`. Not implementation. Verified against source 2026-06-11.

---

## Decision

The Cubric wrapper is a small Python HTTP/WebSocket service running inside the Pod alongside ComfyUI (port 8889, internal). It presents a Cubric-specific surface to the desktop app via the RunPod HTTP proxy, enforces token auth on every interaction including WS upgrades, synthesizes signals that the local deployment derives from process stdout, and proxies the exact capability set of `ComfyUIController` + `commandExecutor` without exposing raw ComfyUI publicly.

This document defines the full endpoint contract. Every section maps to one or more entries in the local-coupling inventory (items 1–10 from `plan.md`) and one or more methods in `comfyController.js` or `commandExecutor.js`.

## Auth Model

### Token Type

A single **Bearer token** (`X-Cubric-Token`) is generated by the desktop app when a Pod is first created and stored via `safeStorage` (see `secret-storage.md`). The same token is injected into the Pod at creation time as the environment variable `CUBRIC_TOKEN`. The wrapper reads it at startup and rejects all requests that do not carry it.

### HTTP Auth

Every endpoint except `GET /health` requires:

```
Authorization: Bearer <token>
```

401 is returned with body `{ "error": "unauthorized" }` if absent or wrong.

### WebSocket Auth

The browser `WebSocket` API does not support custom headers. The WS upgrade URL carries the token as a query parameter:

```
wss://[POD_ID]-8889.proxy.runpod.net/ws?clientId=<uuid>&token=<token>
```

The wrapper validates `token` during the HTTP upgrade handshake (before the protocol switch). An invalid or absent token causes the server to respond with HTTP 401 before the upgrade completes. This means the renderer's `connect()` receives an `onerror` / `onclose` and no WS frames are exchanged. The `token` query parameter is not logged.

### Why Not Mutual TLS or API-Key Header for WS

RunPod's HTTP proxy terminates TLS and forwards plain HTTP/WS to the Pod. Mutual TLS is therefore not available at the application layer. Bearer-in-query for WS is the standard workaround; the token transits TLS between the client and RunPod's proxy terminator, which is acceptable given the token rotation policy defined in `secret-storage.md`.

## Health and Readiness

### `GET /health`

No auth required. Returns immediately.

```json
{ "ready": false, "comfy_ready": false, "wrapper_version": "1.0.0" }
```

`ready` is `true` only when both the wrapper process is up AND internal ComfyUI is responding on `http://127.0.0.1:8188/`. `comfy_ready` tracks internal ComfyUI independently (useful during wrapper warm-up when ComfyUI is still loading). `wrapper_version` is a string matching the Docker image tag.

**Desktop app behaviour:** `GET /remote/status` on Express polls this endpoint every 2 seconds after Pod start until `ready: true` or timeout (240 s, matching `COMFY_READY_TIMEOUT_S` in `comfyController.js:17`). This replaces the local `ensureServerRunning` polling loop.

**Inventory coverage:** Item 8 (boot gate parallel path).

## Synthesized Model-Init Signal (Replaces Stdout Watcher)

**Problem:** Local mode watches ComfyUI's process stdout for `"Model Initializing"` and `"Model Initialization complete!"` strings (`routes/comfy.js:66–77`). The remote Pod's stdout is not visible to the desktop app. `commandExecutor.js:768` subscribes to `GET /comfy/events/stream` (SSE) for these events to drive the `tool:loading-model` badge during terminal-phase sampler workflows.

**Solution:** The wrapper synthesizes equivalent SSE events by observing internal ComfyUI's WebSocket event stream.

### Synthesis Rules

1. **`comfy:model-initializing`** — emitted when the wrapper receives a `executing` event from internal ComfyUI where the executing node is classified as a loader class type (node `class_type` matching `CheckpointLoaderSimple`, `UNETLoader`, `VAELoader`, `CLIPLoader`, `ControlNetLoader`, `LoraLoader`, or any node whose `class_type` ends with `Loader`). The wrapper fetches `GET http://127.0.0.1:8188/object_info` once at startup to build a class-type index.

2. **`comfy:model-init-complete`** — emitted when the same executing-node sequence transitions to a non-loader node, or when a `progress` event with `value > 0` arrives for the terminal-phase sampler node. This mirrors the local pattern where stdout `"Model Initialization complete!"` appears just before sampling begins.

Both events are broadcast on the SSE event stream endpoint below.

### `GET /wrapper/events/stream` (SSE)

Auth required. Content-Type: `text/event-stream`.

Emits the following named events:

| Event name | Data shape | Trigger |
|---|---|---|
| `connected` | `{}` | Immediately after SSE connection established |
| `comfy:model-initializing` | `{ "message": "Model loading..." }` | See synthesis rules above |
| `comfy:model-init-complete` | `{ "message": "Model ready" }` | See synthesis rules above |

Express `GET /comfy/events/stream` in remote mode opens this endpoint server-side, reads the SSE stream, and re-emits the events to the renderer — preserving the `commandExecutor.js:768` subscription path exactly.

**Inventory coverage:** Item 4 (stdout watcher replacement).

## Workflow Submit

### `POST /wrapper/prompt`

Auth required. Body and response are identical to ComfyUI's native `POST /prompt`:

**Request body:**
```json
{
  "prompt": { "<node_id>": { ... } },
  "client_id": "<uuid>"
}
```

The `prompt` field is the fully compiled workflow JSON after Cubric's title-based injection (performed by `commandExecutor.js` before this call — the wrapper receives an already-injected workflow). The wrapper does not re-inject parameters; it validates the workflow bundle version and forwards to internal ComfyUI.

**Compatibility check:** The wrapper reads `CUBRIC_WORKFLOW_BUNDLE_VERSION` from its environment and compares against the `_cubric_bundle_version` field present in the injected workflow (added by a future workflow-bundle versioning step — see Phase 3). If versions are incompatible, the wrapper returns:
```json
{ "error": "bundle_incompatible", "required": "1.2.0", "got": "1.0.0" }
```
with HTTP 409. The desktop app surfaces this as a "Pod needs update" repair path.

**Success response (forwarded from ComfyUI):**
```json
{ "prompt_id": "<uuid>", "number": 1, "node_errors": {} }
```

Express `POST /proxy/prompt` forwards to this endpoint and returns the response verbatim to the renderer. `comfyController.js:501` sends `POST http://${serverAddress}/prompt` → Express → wrapper → internal ComfyUI. The `prompt_ack` synthetic event is emitted on the WS channel by the wrapper once `prompt_id` is confirmed.

**Inventory coverage:** Item 1 (prompt submit), item 3 (prompt_ack synthetic event). Does not bypass Cue queue — Cubric's `generationService` Cue queue operates client-side before this call; one prompt is in flight at a time by design.

## WebSocket Event Channel

### `GET /ws?clientId=<uuid>&token=<token>` (WebSocket Upgrade)

Auth required via `token` query parameter (validated during upgrade, see Auth Model above).

After upgrade the wrapper opens a parallel WS connection to internal ComfyUI at `ws://127.0.0.1:8188/ws?clientId=<uuid>` (same `clientId`). All frames received from internal ComfyUI are forwarded to the client connection. The wrapper adds synthesis logic for `prompt_ack` and model-init signals.

#### Text Frame Contract (JSON events)

The wrapper forwards the following event types verbatim from internal ComfyUI to the client. These are the exact types handled by `commandExecutor.js:779–881`:

| Type | Source | Notes |
|---|---|---|
| `execution_cached` | Internal ComfyUI | Forwarded verbatim. `commandExecutor.js:785` checks `_hasSeedNode` to decide whether to treat as a cache hit. |
| `executing` | Internal ComfyUI | Forwarded verbatim. `node: null` signals completion. `node: <id>` signals active node. |
| `executed` | Internal ComfyUI | Forwarded verbatim. Contains `output.images` / `output.gifs` / `output.latents`. |
| `progress` | Internal ComfyUI | Forwarded verbatim. `{ data: { value, max, node, prompt_id } }`. |
| `progress_state` | Internal ComfyUI | Forwarded verbatim. Node-level state map. |
| `prompt_ack` | Synthesized by wrapper | Emitted once after `POST /wrapper/prompt` receives a `prompt_id` from internal ComfyUI. Shape: `{ type: "prompt_ack", prompt_id: "<uuid>" }`. |

The wrapper does NOT suppress or transform any of these events. `commandExecutor.js` must receive them unchanged to preserve progress aggregation, cache-hit detection, and completion signalling.

#### Binary Frame Contract (Latent Preview Frames)

Internal ComfyUI emits binary `ArrayBuffer` frames: 8-byte header + JPEG payload (TAESD preview, enabled via `--preview-method taesd` at ComfyUI launch — `routes/comfy.js:245` equivalent in the Pod startup command).

The wrapper forwards binary frames to the client **without modification**. The 8-byte header format and JPEG body are preserved exactly so `comfyController.js:270–297` can slice and create a blob URL unchanged.

**Frame relay path:** Internal ComfyUI WS → wrapper WS handler (binary passthrough) → RunPod proxy TLS → renderer `_ws.onmessage (event.data instanceof ArrayBuffer)` → `event.data.slice(8)` → `new Blob([...], { type: 'image/jpeg' })` → `URL.createObjectURL(blob)` → `onPreview(url)`.

**Inventory coverage:** Items 1, 2, 3 (all event types including binary frames).

## Output Fetch

### `GET /wrapper/view`

Auth required. Query parameters are identical to ComfyUI's native `GET /view`:

```
/wrapper/view?filename=<f>&type=<t>&subfolder=<s>&format=<fmt>
```

The wrapper validates the token, then proxies the request to `http://127.0.0.1:8188/view?...` and streams the response body to the client. No caching, no transformation.

Express `GET /proxy/view` attaches the token and forwards to this endpoint. View URLs in the renderer (`commandExecutor.js:33–49`) point at `http://127.0.0.1:3000/proxy/view?...` — localhost Express, no RunPod URL visible to renderer. `routes/projects.js:1398` calls `streamDownload(comfyViewUrl, filePath)` against the Express URL; Express adds auth and streams from wrapper.

**Why auth on `/view`:** Without auth, any party who knows the Pod ID could fetch generated outputs by constructing a RunPod proxy URL directly (since Pod IDs are guessable given an account). Auth gating prevents output leakage.

**Inventory coverage:** Item 6 (output capture, view URL, save-generation streaming).

Auth required. Body forwarded verbatim to internal ComfyUI `POST /interrupt`:

Returns `{}` on success. `comfyController.js:150` maps to Express `POST /proxy/interrupt` → wrapper.

## Upload and Staging Endpoints

All upload endpoints are auth required.

### `POST /wrapper/upload/image`

Accepts `multipart/form-data` with fields matching ComfyUI's native `POST /upload/image`:

- `image` — file blob (PNG/JPEG/WebP)
- `overwrite` — `"true"` or `"false"`

The wrapper forwards the multipart form to `http://127.0.0.1:8188/upload/image` and returns the response verbatim:
```json
{ "name": "mpi_input_image.png", "subfolder": "", "type": "input" }
```

The static-filename convention (`mpi_<param>.png`) is enforced by the caller (`comfyController.js:386`) not the wrapper. The wrapper accepts any filename from `overwrite: true` requests.

Express `POST /upload/image` in remote mode forwards to this endpoint with auth attached. Renderer `comfyController.js:558` calls `POST http://${serverAddress}/upload/image` → Express → wrapper → internal ComfyUI. The image/mask blob upload path is unchanged from the renderer perspective.

**Inventory coverage:** Item 5 (image/mask upload).

### `POST /wrapper/upload/media`

Accepts `multipart/form-data`:

- `file` — video or audio file blob
- `filename` — target filename (basename only; wrapper rejects paths containing `/` or `..`)
- `overwrite` — `"true"` or `"false"` (default `"true"`)

The wrapper writes the file to `/workspace/comfyui/input/<filename>` (where ComfyUI's `input` directory is a symlink or configured path on the volume). Returns:
```json
{ "name": "<filename>", "type": "input", "path": "/workspace/comfyui/input/<filename>" }
```

This replaces local path injection (`_resolveMediaPath`, `comfyController.js:574–596`). In remote mode, `commandExecutor.js` resolves media to an absolute local path, then calls Express `POST /remote/upload/media` (which streams the local file to this endpoint) before building the workflow param map. The workflow node receives the bare filename rather than an absolute path; the wrapper's ComfyUI launch args configure `input_directory` so `input/<filename>` is the correct path-loader target.

**VHS node note:** WAN/CogVideo VHS `LoadVideo` nodes accept `filename` (relative to `input/`) not an absolute path when `input_directory` is configured. The wrapper's ComfyUI launch config must set `--input-directory /workspace/comfyui/input` so the path convention works.

**Inventory coverage:** Item 5 (video/audio path injection replacement).

### `POST /wrapper/upload/latent`

Accepts `multipart/form-data`:

- `file` — `.latent` file blob
- `filename` — target filename (basename only)

Writes to `/workspace/comfyui/input/<filename>`. Returns:
```json
{ "name": "<filename>", "type": "input" }
```

This replaces `POST /comfy/stage-preview-latent` (`routes/comfy.js:170–195`) in remote mode. Express intercepts `/comfy/stage-preview-latent` when `remoteEngineActive`, reads the local latent file from `sourcePath`, and POSTs it to this endpoint. The `engineInputName` param maps directly to `filename`.

**Workflow input defaults:** `_prepareWorkflowInputs` in `commandExecutor.js:58–73` calls `POST /comfy/prepare-workflow-inputs` to copy default latents (e.g. `ComfyUI_00001_.latent`) into the ComfyUI `input/` folder before `_ms` workflows. In remote mode, Express `POST /comfy/prepare-workflow-inputs` uploads the default latent files (shipped with the app under `comfy_workflows/input/`) to the wrapper via `POST /wrapper/upload/latent`. The wrapper writes them to `/workspace/comfyui/input/` idempotently.

**Inventory coverage:** Item 5 (latent staging).

## Model and Dependency Status

### `GET /wrapper/models/status`

Auth required. Body: same shape as local `POST /comfy/models/check` — the desktop app sends the same payload:

```json
{
  "models": [
    { "id": "sdxl_base", "deps": [{ "type": "checkpoints", "filename": "sd_xl_base_1.0.safetensors", "id": "sdxl_base_ckpt" }] }
  ]
}
```

Note: this endpoint accepts a request body via GET or can be expressed as `POST /wrapper/models/status` — using POST to carry the body is conventional; the Express interceptor sends POST.

The wrapper resolves filenames against `/workspace/mpi_models/` (the volume model store, equivalent to local `extra_model_paths.yaml`'s `base_path`). It uses the same sub-directory conventions (checkpoints, loras, upscale_models, etc.) and applies the same `isCompleteOnDisk` logic (checks file exists and size matches). Returns the same shape as local `POST /comfy/models/check`:

```json
{
  "success": true,
  "results": {
    "sdxl_base": {
      "installed": true,
      "deps": [{ "id": "sdxl_base_ckpt", "installed": true, "partialBytes": 0 }]
    }
  }
}
```

Express intercepts `POST /comfy/models/check` in remote mode and forwards to this endpoint. The rest of the model-check call chain (model registry, progress computation, UI) is unchanged.

**Inventory coverage:** Item 9 (model/dependency status on volume).

## Lifecycle Error Mapping

The wrapper uses a consistent error envelope on all non-2xx responses:

```json
{ "error": "<error_code>", "message": "<human-readable>", "detail": {} }
```

| HTTP status | `error` code | Meaning |
|---|---|---|
| 401 | `unauthorized` | Missing or invalid token (HTTP) |
| 409 | `bundle_incompatible` | Workflow bundle version mismatch |
| 409 | `comfy_busy` | ComfyUI already executing a prompt (should not happen under Cubric's single-dispatch Cue queue, but defensive) |
| 422 | `invalid_payload` | Malformed request body |
| 503 | `comfy_not_ready` | Internal ComfyUI is not yet responding |
| 503 | `engine_error` | ComfyUI returned an error during prompt execution |
| 500 | `internal_error` | Unexpected wrapper failure |

Express translates these codes into app-level error events (`Events.emit('ui:error', ...)`) using a mapping table in `remoteProxy.js`.

## ComfyUIController Capability Map

The following table proves the 1:1 mapping from every existing `ComfyUIController` method and `commandExecutor` capability to the wrapper surface, without bypassing Cubric's Cue queue or title-based workflow injection.

| Local capability | Local call site | Remote path | Wrapper endpoint | Auth |
|---|---|---|---|---|
| Prompt submit | `comfyController.js:501` `POST http://serverAddress/prompt` | Renderer → Express `POST /proxy/prompt` → wrapper | `POST /wrapper/prompt` | Bearer token |
| WS connect (JSON) | `comfyController.js:291` `new WebSocket(ws://serverAddress/ws)` | Renderer → Express WS relay → wrapper | `GET /ws?clientId&token` | Token in query |
| WS connect (binary preview) | `comfyController.js:270–297` | Renderer WSS-direct (exception) | `GET /ws?clientId&token` | Token in query |
| Interrupt | `comfyController.js:152` `POST /interrupt` | Renderer → Express `POST /proxy/interrupt` → wrapper | `POST /wrapper/interrupt` | Bearer token |
| Clear queue | `comfyController.js:192` `POST /queue {clear:true}` | Renderer → Express `POST /proxy/queue` → wrapper | `POST /wrapper/queue` | Bearer token |
| Delete queue item | `comfyController.js:211` `POST /queue {delete:[id]}` | Renderer → Express `POST /proxy/queue` → wrapper | `POST /wrapper/queue` | Bearer token |
| Image/mask upload | `comfyController.js:558` `POST /upload/image` | Renderer → Express `POST /upload/image` intercept → wrapper | `POST /wrapper/upload/image` | Bearer token |
| Video/audio path injection | `comfyController.js:382` `_resolveMediaPath` | commandExecutor uploads local file via Express `POST /remote/upload/media` → wrapper; workflow receives bare filename | `POST /wrapper/upload/media` | Bearer token |
| Latent staging | `commandExecutor.js:91–106` `POST /comfy/stage-preview-latent` | Express intercept in remote mode reads local file → uploads → wrapper | `POST /wrapper/upload/latent` | Bearer token |
| Workflow input defaults | `commandExecutor.js:58–73` `POST /comfy/prepare-workflow-inputs` | Express uploads bundled default latents → wrapper | `POST /wrapper/upload/latent` | Bearer token |
| Output fetch (view URL) | `commandExecutor.js:33–49` builds `http://serverAddress/view?...` | View URL → Express `GET /proxy/view` → wrapper | `GET /wrapper/view` | Bearer token |
| ComfyUI status / readiness | `comfyController.js:67` `GET /comfy/status` | Express `GET /remote/status` polls `/health`; local `/comfy/status` remains for local mode | `GET /health` | None (health only) |
| Model-init SSE | `commandExecutor.js:768` `new EventSource('/comfy/events/stream')` | Express `/comfy/events/stream` relays from wrapper SSE | `GET /wrapper/events/stream` | Bearer token (Express adds) |
| Model status check | `routes/comfy.js:409` `POST /comfy/models/check` | Express intercept → wrapper | `POST /wrapper/models/status` | Bearer token |

**Cue queue preservation:** Cubric's `generationService` enforces single-dispatch: one `runWorkflow` call is in flight at a time. The wrapper receives compiled, injected workflow JSON — it does not perform title-based injection or Cue scheduling. Those remain client-side. The wrapper's `comfy_busy` 409 is a defensive guard, not a queue implementation.

**Title-based injection preservation:** `commandExecutor.js` resolves `_resolveWorkflowFile`, applies `INJECTORS`, and builds the final `params` map before calling `ComfyUIController.runWorkflow`. The workflow JSON reaching `POST /wrapper/prompt` is fully injected. The wrapper is a transparent carrier.
