# MPI-256 — Artistic tier: App Library + Apps (Higgsfield-style outcome apps)

## Current State

Project mode: **scalable-foundation** — full guardrails, no prototype shortcuts. All product/architecture decisions were front-loaded and locked with the user in [design.md](design.md) §1–13; investigation is complete in [research/](research/) (7 reports + SYNTHESIS.md). **Read design.md §12 + research/SYNTHESIS.md before implementing anything.**

Locked essentials:
- **Naming law:** Workspaces stay Landing/Gallery/History. "App Library" + "App" are OVERLAYS. Never "app workspace".
- Flow: Gallery → dev-gated toolbar button → App Library overlay (grid + availability badges) → card click → detail slide-over (info + required models + Open/Install) → App Library closes → App overlay opens (Block + minimal controls) → Run → job enters the EXISTING queue.
- Registry: `js/data/apps.js`; descriptor `{id, title, preview, description, requiredModels: [MODEL ids], operation, workflow, uiComponent, inputSchema}`.
- Run model: app = second producer via `enqueueGeneration()` (generationService.js:380), pattern = History block's universal tool ops (`model: {id:null, mediaType}`, MpiGroupHistoryBlock.js:1116-1173). NO `getNextGeneration` callback. Run-clean LoRA policy.
- Results = normal gallery cards. Sidecar gets ADDITIVE TOP-LEVEL `appId` + `appInputs`; Reuse on app cards opens the App with inputs restored (branch at `_applyPromptReuse` top, both blocks).
- New MpiOverlay mount mode **`mountTarget: 'main-area'`** (covers #tool-container + #prompt-box-mount, spares status bar). Queue slide-over z fix via `--app-overlay-z` CSS var.
- Dev-gated by `APP_CONFIG.dev_mode` until ≥4 apps exist. First app = image-in→image-out, 1 model.

Key risks the investigation surfaced (full list research/SYNTHESIS.md § traps): silent Input_* drop (MPI-242), Output_* title law (MPI-217), universal ops have NO model guard (our pre-flight guard is new), hot-store skip for universal ops (accepted for v1 — first app uses a small model), `Overlays.reset()` on navigate force-closes overlays (s_appInputs lives in state.js for that reason), never `renderList()` on hot download events, `payload.item` must stay intact through reuse.
