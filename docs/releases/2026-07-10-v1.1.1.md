# Cubric Vision v1.1.1 — 2026-07-10

Patch release on the 1.1.0 line. Three local-engine fixes surfaced while
verifying the 1.1.0 build on a clean Windows machine (the local engine had not
run on ComfyUI 0.27 before shipping). No new operations, no engine change, no
schema change.

## Changelog

### Fixes
- **Missing-LoRA generation failure is now clear and actionable.** When a
  generation used a LoRA that is not in your folders (e.g. on a clean install, or
  after removing the folder in Settings), it failed with the crash-report dialog
  and left the Cue queue stuck on a RUNNING card whose Stop button did nothing —
  which read as "generation is broken." It now names the missing LoRA and how to
  add it, and the queue card clears. Add the LoRA (or pick another) and generation
  runs. This is the same fix as the local /prompt validation error surfacing as a
  crash dialog.
- **Auto-mask works on Windows again.** The Detect button in the Mask tool no
  longer fails on the Windows local engine with a model-path (`value not in
  list`) error. The path-separator heal was corrupting the Impact Pack detector's
  self-listed `bbox/face_yolov8n.pt` model name.
- **Fresh engine setup is resilient.** First-time engine setup no longer strands
  later custom-node dependencies when one node's install hits a transient error
  mid-batch (previously it aborted the whole batch, leaving components
  uninstalled until the user pressed Retry), and no longer leaves behind
  duplicate download files (e.g. `RES4LYF (1).zip`).

### New Operations
None.

### Breaking Changes
None.

### ComfyUI Engine
ComfyUI version unchanged (0.27.0).
