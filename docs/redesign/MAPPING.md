## 3. Var fallback elimination

Remove the hex fallback from every `var(--X, #hex)` expression. The compat alias block in Phase 0.1 ensures legacy var names resolve correctly — the fallback is redundant and will mask sweep misses.

| Current | Replace with |
|---|---|
| `var(--neon-electric, #aa8bcd)` | `var(--accent-heat)` |
| `var(--primary, #9a82bb)` | `var(--accent-heat)` |
| `var(--text-primary, #fff)` | `var(--ink-1)` |
| `var(--surface-0, #0d0d0f)` | `var(--surface-0)` |
| `var(--text-1, #e8e6f0)` | `var(--ink-1)` |
| `var(--danger, #ff4d4d)` | `var(--accent-heat)` |
| `var(--neon-electric, #7df)` | `var(--accent-heat)` |
| `var(--surface-1, #000)` | `var(--surface-1)` |
| `var(--neon-warm, #f90)` | `var(--accent-warn)` |

| Old value | Nearest token | Token value |
|---|---|---|
| `10px` | `--t-2xs` | 10px |
| `11px`, `0.75rem` | `--t-xs` | 11px |
| `12px` | `--t-xs` | 11px (closest — round down) |
| `13px`, `0.8rem` | `--t-sm` | 13px |
| `14px`, `0.875rem` | `--t-sm` | 13px (closest) |
| `15px`, `0.9rem` | `--t-md` | 15px |
| `1rem` (base 16px) | `--t-md` | 15px |
| `16px` | `--t-md` | 15px |
| `18px`, `1.1rem` | `--t-lg` | 19px |
| `19px`, `1.2rem` | `--t-lg` | 19px |

## 5. Spacing scale

Convert px / rem values to the nearest spacing token. Prefer exact matches; use adjacent token when gap is small (≤2px). For structural layout values like `180px` bottom padding, leave literal — not all spacing is rhythmic.

| Old value | Token | Token value |
|---|---|---|
| `4px`, `0.25rem` | `--s-1` | 4px |
| `8px`, `0.5rem` | `--s-2` | 8px |
| `10px`, `0.6rem` | `--s-2` | 8px (close enough) |
| `14px`, `0.9rem` | `--s-3` | 14px |
| `16px`, `1rem` | `--s-3` | 14px (closest) |
| `20px`, `1.25rem` | `--s-3` | 14px (or leave — structural) |
| `22px`, `1.4rem` | `--s-4` | 22px |
| `24px`, `1.5rem` | `--s-4` | 22px (closest) |
| `32px`, `2rem` | `--s-5` | 32px |

**Shell CSS note:** The shell CSS (landing, workspace) uses `rem` units extensively. These are being replaced in Phase 5.1 (Landing port) and Phase 8.1 (cross-cutting), not in the Phase 2 sweep. Phase 2 is component CSS only.

## 6. Radius scale

| Old value | New value | Rule |
|---|---|---|
| `0px`, `border-radius: 0` | `var(--r-1)` | sharp — Stage default |
| `2px` | `var(--r-2)` | closest token (4px) — check if must be 2px; if so keep literal |
| `4px` | `var(--r-2)` | exact |
| `6px` | `var(--r-2)` | closest (round down from 4px side) |
| `8px` | `var(--r-2)` | closest (4px) — or `--r-3` if context is a container |
| `10px` | `--r-3` | closest (12px) |
| `12px` | `var(--r-3)` | exact |
| `16px` | `var(--r-3)` | closest (12px) |
| `50%` | keep literal | circle semantics — do not tokenize |
| `999px`, `100px`, `--r-pill` | keep literal or `var(--r-pill)` | pill semantics |

**Note:** `MpiMemoryMonitor` `border-radius: 2px` on gauge bars — keep literal (2px is intentionally tighter than `--r-2`; the gauge bar is a 1px-tall element per DESIGN.md spec).

## 7. Motion — transition replacement

**Banned:** any `cubic-bezier` that isn't Stage's `--ease`. Drop springy/bounce cubics entirely.

| Old pattern | New value |
|---|---|
| `0.1s ease`, `0.12s ease` | `var(--t-fast) var(--ease)` (200ms) |
| `0.15s ease` | `var(--t-fast) var(--ease)` |
| `0.2s ease` | `var(--t-fast) var(--ease)` |
| `0.25s ease`, `0.3s ease` | `var(--t-base) var(--ease)` (280ms) |
| `0.4s ease` | `var(--t-slow) var(--ease)` (480ms) |
| `0.6s cubic-bezier(0.4, 0, 0.2, 1)` | `var(--t-slow) var(--ease)` |
| `0.3s cubic-bezier(0.4, 0, 0.2, 1)` | `var(--t-base) var(--ease)` |
| `0.4s cubic-bezier(0.175, 0.885, 0.32, 1.275)` | `var(--t-slow) var(--ease)` — **drop the overshoot** |
| `transition: all ...` | Replace `all` with specific properties (`background`, `color`, `opacity`, `transform`) — never animate layout |
| Any `ease-in` / `linear` | Replace with `var(--ease)` — Stage is always ease-out |

## 9. JS canvas color literals (wave 2.6 scope)

Canvas draws happen per-frame; JS cannot use `var(--token)` without `getComputedStyle` ceremony. Hardcode OKLCH hex constants at the top of each file with a comment referencing the token.

| Role | JS constant | OKLCH value | Token reference |
|---|---|---|---|
| Mask overlay gradient stop 1 (60% alpha) | `MASK_COLOR_60` | `oklch(0.72 0.20 6 / 0.6)` | `--accent-heat` |
| Mask overlay gradient stop 2 (25% alpha) | `MASK_COLOR_25` | `oklch(0.72 0.20 6 / 0.25)` | `--accent-heat` |
| Mask outline (dashed) | `MASK_OUTLINE` | `oklch(0.72 0.20 6 / 0.8)` | `--accent-heat` |
| Crop frame stroke | `CROP_STROKE` | `oklch(0.66 0.014 80)` | `--ink-3` |
| Brush cursor fill | `BRUSH_CURSOR` | `oklch(0.72 0.20 6)` | `--accent-heat` |

**Mask radial gradient spec (CSS canvas overlay):**
```css
radial-gradient(ellipse at center,
  color-mix(in oklch, var(--accent-heat) 60%, transparent) 0%,
  color-mix(in oklch, var(--accent-heat) 25%, transparent) 60%,
  transparent 78%)
```

**For JS canvas `fillStyle`:**
```js
/* --accent-heat at 60% alpha */
const MASK_COLOR_60 = 'oklch(0.72 0.20 6 / 0.6)';
const MASK_COLOR_25 = 'oklch(0.72 0.20 6 / 0.25)';
const MASK_OUTLINE  = 'oklch(0.72 0.20 6 / 0.8)';
const CROP_STROKE   = 'oklch(0.66 0.014 80)';     /* --ink-3 */
const BRUSH_CURSOR  = 'oklch(0.72 0.20 6)';        /* --accent-heat */
```

Note: Modern Chromium (Electron 41+) supports OKLCH in canvas `fillStyle`/`strokeStyle`. No hex conversion needed.

## 10. Banned patterns reminder

No sub-agent may introduce these patterns in Phase 2:

- **No hex** outside this mapping doc (`#xxx`, `#xxxxxx`). Zero exceptions.
- **No named CSS colors** (`red`, `white`, `black`, `transparent` is OK in masks only).
- **No raw `rgb()` / `rgba()`** in component CSS. Use `color-mix(in oklch, ...)` or OKLCH with `/` alpha.
- **No `backdrop-filter`** unless it's the canvas dock backdrop (Phase 4 only).
- **No gradient text** outside the `.wordmark` selector.
- **No neon glow** box-shadows using the old neon rgba values.
- **No BEM class renames** — CSS-only sweep, classes stay identical.
- **No markup changes** — Phase 2 touches `.css` files only (except wave 2.6 which touches `.js` files).
- **No `js/components/types.js` edits** in Phase 2.
- **No `preloadStyles.js` edits** in Phase 2.
