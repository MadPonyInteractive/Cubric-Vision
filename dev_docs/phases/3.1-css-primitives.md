## Context

Component-specific CSS rules currently live inside `styles/03_forms.css` mixed with global form styles. The factory's `ensureStylesheet()` already handles deduplication, so once a rule lives inside a component's `.css` file and is referenced in its `css: []` array, it's loaded automatically.

This phase extracts rules for all 8 Primitives. We'll grep for each BEM namespace, move the block, and delete from the global partial.

---

## Process (repeat for each component)

1. Search for the component's BEM namespace in `styles/03_forms.css` and `styles/04_comfy.css` and `styles/05_tools.css`
2. Copy those rules into the component's `.css` file
3. Verify the component's `.js` factory definition has `css: ['js/components/Primitives/ComponentName/ComponentName.css']`
4. Delete the extracted rules from the global partials
5. Test in browser — navigate to Component Gallery and confirm visuals unchanged

## Rules for What to Move vs Keep Global

**MOVE to component CSS:**
- Any rule whose selector starts with `.mpi-{component}` or `.mpi-{component}__` or `.mpi-{component}--`
- Any rule that ONLY applies to that component

**KEEP in global `styles/`:**
- Rules that affect multiple components or page layout
- CSS custom properties / design tokens (`--color-*`, `--radius-*`)
- Rules that reference both a component class AND a layout class (e.g., `.tool-header .mpi-btn`)
