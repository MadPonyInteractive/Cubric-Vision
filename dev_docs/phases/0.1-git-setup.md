# Phase 0.1 — Git Initialization

> **Session type:** Foundation  
> **Estimated time:** 15 min  
> **Files touched:** root only (no JS, no CSS)  
> **Depends on:** nothing — run this first

---

## Context

MpiAiSuite has no git repository. This phase initializes one with a proper structure.
The user will connect a remote GitHub account separately after setup.

---

## Tasks

### 1. Initialize git

```powershell
cd c:\AI\Mpi\MpiAiSuite
git init
git checkout -b dev
```

Working branch is `dev`. `main` is for stable snapshots only.

---

### 2. Create `.gitignore`

Create `c:\AI\Mpi\MpiAiSuite\.gitignore` with this exact content:

```
# Dependencies
node_modules/

# Electron build output
dist/
out/

# Engine binaries and embedded Python (huge, gitignored)
engine/
llama_engine/

# User data
projects/
data/

# Temporary files
tmp/
*.tmp
*.log

# OS artifacts
.DS_Store
Thumbs.db
desktop.ini

# Editor
.vscode/settings.json
*.code-workspace

# Secrets / local config (if any added later)
.env
.env.local

# Backup folder (manual snapshot, not version controlled)
Backup/
```

---

### 3. Initial commit

```powershell
git add .
git commit -m "[R8-0.1] Initial commit — pre-refactor snapshot"
```

---

### 4. Create branch strategy comment in `dev_docs/06_refactor_master_plan.md`

Already documented — no changes needed.

## Verification

- [ ] `git status` shows clean working tree after commit
- [ ] `.gitignore` is committed
- [ ] `git branch` shows `dev` as current branch
- [ ] `node_modules/`, `engine/`, `projects/` are NOT tracked (`git ls-files` check)

---

## What NOT to do

- Do NOT push to any remote (user will set that up separately)
- Do NOT change any JS or CSS files
- Do NOT create a `main` branch manually — `git checkout -b dev` is enough for now
