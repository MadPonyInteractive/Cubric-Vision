## Step 1 — Fix `appendMessage()` inline styles

The current `appendMessage()` function uses heavy inline styles (background colors, padding, border-radius etc.). These belong in CSS.

Open `styles/05_tools.css` and check if `.chat-message`, `.chat-message.user`, `.chat-message.assistant` classes already exist.

**If they exist:** Remove the inline styles from `appendMessage()` and rely on the classes.

**If they don't exist:** Add them to `styles/05_tools.css`:
```css
/* LLM Chat messages */
.chat-message {
    max-width: 85%;
    padding: 0.85rem 1rem;
    border-radius: 12px;
    margin-bottom: 0.5rem;
    white-space: pre-wrap;
    box-shadow: 0 2px 8px rgba(0,0,0,0.15);
    font-size: 0.95rem;
    line-height: 1.5;
}
.chat-message.user {
    align-self: flex-end;
    background-color: var(--primary-color);
    color: #fff;
    border-bottom-right-radius: 2px;
}
.chat-message.assistant {
    align-self: flex-start;
    background-color: var(--bg-lighter);
    border-bottom-left-radius: 2px;
    border: 1px solid var(--border-color);
}
```

Then simplify `appendMessage()` to only set the `className` and `textContent` — remove all `msgDiv.style.*` lines.

> ⚠️ Only move styles that are NOT already present in the CSS file. Don't duplicate.
