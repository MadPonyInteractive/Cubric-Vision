### 2. Add Event Bus section

```markdown
## 📡 Event Bus (`js/events.js`)

> **Rule:** Use `Events.emit/on` for cross-module communication. Never `document.dispatchEvent` in tool files.

```js
import { Events } from '../events.js';

// Subscribe
const unsub = Events.on('media:updated', ({ projectId }) => refresh(projectId));
unsub(); // cleanup

// One-time
Events.once('comfy:ready', () => startRun());

// Emit
Events.emit('media:updated', { projectId: state.currentProject.id });

// Namespaced channel (for tool-scoped events)
const ch = Events.channel('generator');
ch.emit('result', { url });
ch.on('result', handler);
```

### Canonical Event Names
| Event | Payload | Who emits |
|---|---|---|
| `media:updated` | `{ projectId }` | Any tool saving to library |
| `tool:running` | `{ tool, type }` | `toolUtils.setRunningTool()` |
| `tool:idle` | `{ tool, type }` | `toolUtils.clearRunningTool()` |
| `project:changed` | `{ project }` | `shell.js` |
| `state:changed` | `{ key, value }` | `state.js` |
| `comfy:ready` | — | `comfyController.js` |
| `nav:tool` | `{ toolName }` | `router.js` |
| `templates:updated` | `{ projectId }` | `toolState.js` |
```

### 3. Add Project Templates section

```markdown
## 📋 Project Templates (`js/toolState.js`)

Users can save full project state snapshots and restore them instantly.

```js
import { saveTemplate, loadTemplate, listTemplates, deleteTemplate } from '../toolState.js';

// Save current state of all tools
await saveTemplate('Sharp Portrait Style');

// Restore a saved template (writes all tool states to localStorage)
await loadTemplate('Sharp Portrait Style');

// List all saved templates for active project
const names = await listTemplates(); // ['Sharp Portrait Style', 'Cinematic Look']

// Delete
await deleteTemplate('Sharp Portrait Style');
```

Templates are stored in `project.json` under `"templates": {}`. They survive browser cache clears.
Backend routes: `GET/POST /project-templates/:id`, `DELETE /project-templates/:id/:name`.
```
