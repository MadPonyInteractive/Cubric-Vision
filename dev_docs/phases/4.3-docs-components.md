## 🔌 Tool Consumer Pattern

When rebuilding a tool to use factory components:

```js
// 1. Import what you need
import { MpiMediaDropzone } from '../components/Compounds/MpiMediaDropzone/MpiMediaDropzone.js';
import { MpiIconButton }    from '../components/Compounds/MpiIconButton/MpiIconButton.js';
import { MpiProgressBar }   from '../components/Primitives/MpiProgressBar/MpiProgressBar.js';
import { MpiToast }         from '../components/Primitives/MpiToast/MpiToast.js';
import { Events }           from '../events.js';

// 2. Mount components into your template's mount points
const dropzone = MpiMediaDropzone.mount(
    document.getElementById('upscaler-dropzone'),
    { title: 'Drop Image', mediaType: ['image'] }
);

// 3. React to component events
dropzone.on('drop', async ({ file }) => {
    const result = await uploadImageToProject(file, 'upscaler');
    if (result?.filePath) state.upscalerInputImage = result.filePath;
});

// 4. Emit bus events for cross-module communication
Events.emit('media:updated', { projectId: state.currentProject.id });

// 5. Destroy on navigation
export function destroy() {
    dropzone.destroy();
    progressBar.destroy();
}
```

```markdown
## Using Factory Components in a Tool

All tools MUST use the factory component system. No manual DOM building.

**Mandatory imports for most tools:**
```js
import { MpiPromptBox }     from '../components/Blocks/MpiPromptBox/MpiPromptBox.js';
import { MpiIconButton }    from '../components/Compounds/MpiIconButton/MpiIconButton.js';
import { MpiMediaDropzone } from '../components/Compounds/MpiMediaDropzone/MpiMediaDropzone.js';
import { Events }           from '../events.js';
```

**Mount pattern:** Always mount into a container element defined in the HTML template.
**Destroy pattern:** Always call `instance.destroy()` in the tool's `destroy()` export.
**Events:** Use `Events.emit('media:updated', ...)` not `document.dispatchEvent`.
```
