# Phase 3.3 — CSS Audit: Dead Rules & Global Cleanup

> **Session type:** CSS cleanup  
> **Estimated time:** 45 min  
> **Files touched:** `styles/03_forms.css`, `styles/04_comfy.css`, `styles/05_tools.css`  
> **Depends on:** Phases 3.1 and 3.2

---

## Context

After Phases 3.1 and 3.2 extracted component-specific CSS, the global partials should now only contain:
- App shell, sidebar, and layout rules
- Tool layout skeletons (`.tool-panel`, `.tool-header`, `.source-preview`)
- Global utility classes
- CSS design tokens and animations

This phase audits what remains and removes dead rules.

---

## Tasks

### 1. Search for orphaned component classes in each partial

For each file (`03_forms.css`, `04_comfy.css`, `05_tools.css`):
- Search for any remaining `.mpi-` prefixed rules that should have been moved
- Search for selectors targeting deleted HTML patterns (from tool rebuilds)
- Mark rules as `/* DEAD? */` for review — do NOT delete without confirming

### 2. Verify each flagged rule is truly dead

For each flagged rule:
```powershell
# Check if selector is used anywhere
Select-String -Path "js/**/*.js","templates/**/*.html","styles/**/*.css" -Pattern "class-name-here" -Recurse
```

If zero hits — it's dead. Delete it.

### 3. Document what remains in each file

After cleanup, add a comment block at the top of each partial documenting what it contains:

```css
/**
 * styles/03_forms.css — Global form and modal styles.
 *
 * Contains:
 * - Modal overlays (.modal-overlay, .modal-content)
 * - Generic form fields (.form-group, .form-label)
 * - Provisioning wizard (.wizard-*)
 * - Model card grid (.model-card-*)
 * - Hover bridge utility (.popup-hover-bridge)
 *
 * Does NOT contain:
 * - Component-specific styles (those are co-located in js/components/)
 */
```
