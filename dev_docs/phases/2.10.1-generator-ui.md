## Step 1 — Replace imports

**Delete:**
```js
import { Slider } from '../components/Slider.js';
import { PromptBox } from '../components/PromptBox.js';
import { renderCustomDropdown } from '../components/customDropdown.js';  // if present
```

**Add:**
```js
import { MpiSlider }        from '../components/Compounds/MpiSlider/MpiSlider.js';
import { MpiPromptBox }     from '../components/Blocks/MpiPromptBox/MpiPromptBox.js';
import { MpiRatioSelector } from '../components/Blocks/MpiRatioSelector/MpiRatioSelector.js';
import { Events }           from '../events.js';

```

Remove `generateSeed` from the `uiHelpers.js` import line.

## Step 2 — Replace `PromptBox` mount

Find `new PromptBox({...})` and replace with:
```js
promptBox = MpiPromptBox.mount(container.querySelector('#generator-prompt-wrapper'), {
    toolId: 'generator'
});
promptBox.on('image-drop', async ({ item }) => {
    // Move the existing onImageDrop handler body here
});
```

> ⚠️ Read `MpiPromptBox` source to confirm the exact event name for image drops.

---

## Step 3 — Replace `Slider` instances

Find each `new Slider(...)` call. There are likely 2–4 (steps, CFG, denoise, etc.).

For each one, replace with `MpiSlider.mount(container, props)` and wire the `change` event.

Keep a reference to each slider instance (e.g., `stepsSlider`, `cfgSlider`) so you can call `.update()` and `.destroy()` later.

> ⚠️ Check the actual prop names in `MpiSlider.js`. Use `min`, `max`, `step`, `value`, `label`.  
> The old `Slider` used `onChange: (v) => {}`. The new one uses `.on('change', ({ value }) => {})`.
